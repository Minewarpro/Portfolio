● Time winding system

Date change

Le joueur peut simplement augmenter ou baisser la date en appuyant sur les fleches haut ou bas. Il peut ensuite voyager a la date choisi.

Les dates disponibles sont enregistrer dans un tableau qui sont sauvegarder pour permettre au joueur de quitter le jeu sans crainte.

● Mini Games

Magic coloring

Pour ce mini jeu le joueur doit colorier les cases avec les couleurs corespondantes. Chaque couleur est associé a une date.

Un raycast est envoyer depuis la camera du joueur jusqu'a l'endroit ou le joueur a cliqué. Une box de detection ce cache derrière chaque partie du dessin pour receptionner ce raycast.

Une fois le coloriage completer avec les bonnes couleur le joueur remporte le mini jeu !

● A little bit of easy fun ?

The player has the ability to grab any physical element, including their ball. We thought that throwing the ball to break things was a good idea.

In the game, all elements with the weathered wood texture can be broken by sending our ball towards them.

It is also possible to grab crates to throw and break them.

● The Player

Managing the player was also a challenge. In Unreal Engine, the default character is derived from the "Character" class, which uses "Character Movement". The problem with this is that it ignores physical constraints that are not predefined within it. Unfortunately, the system mentioned earlier did not work in this case.

The solution I found is to use the parent class "Pawn" and employ another movement management system, "FloatingPawnMovement", which is much more permissive.

The counterparts

The counterpart of this is that there are many features that don't exist, and we have to code them ourselves.

For example, we could mention the management of gravity, the Jump function, or even the On Landed event, which allows the detection of the player when they touch the ground.
