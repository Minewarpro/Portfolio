Comics Project

Software Used Unreal Engine 5
Languages Used Blueprint

The goal of this project was to understand how the new motion matching system works in Unreal Engine 5.

To fully test the system, I wanted to use animations with a 'low frame rate' style.

● Demonstration

Motion Matching

The first step was to import animations from Unreal’s Lyra project and integrate them into my own.

I then set up the motion matching system to achieve smooth animations with a character that constantly looks toward the cursor.

To create a 'comic book' effect, I had to simulate low frame rate animations using a Blueprint, rather than applying it directly to the animations, in order to preserve root motion behavior.

All that’s left is to add a shader to make the whole thing more immersive !

This table allows for managing animation selection based on any number of variables.

MetaHuman

To take the animated character even further, I decided to integrate a Metahuman.

From there, it's just a matter of retargeting the mannequin's animations to the Metahuman using Unreal's built-in tools.

Buoyancy Project

Software Used Unreal Engine 5
Languages Used Bluprint

My goal was to explore Unreal’s 'Water' plugin in order to improve my prototyping efficiency when working with water-based gameplay.

To test it, I built a boat navigation system set in the open sea.

● Buoyancy System

Floating Objects

The initial step was implementing object flotation on water using the Buoyancy Components provided by the 'Water' plugin.

Boat Driving

Next, I implemented a simple boat control system.

While it’s effective for rapid prototyping, the Buoyancy system isn’t well-suited for gameplay occurring on the boat itself, which results in instability for this type of feature.
